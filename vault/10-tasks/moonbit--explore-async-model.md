# Explore the async model in MoonBit

## Outcome

Clarify what async should mean in MoonBit and which tradeoffs seem acceptable around syntax, runtime model, effects, and developer ergonomics.

## Notes

This is a language-design exploration task rather than a direct implementation task. The useful output is a sharper model of whether async belongs as a core language concept, a library pattern, or some hybrid approach.
